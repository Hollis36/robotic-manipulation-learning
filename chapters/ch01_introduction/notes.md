# Chapter 1 Notes

## 本章核心问题

机器人操作为什么远不止“抓起一个物体再放下”？我需要从感知、规划、控制、接触和开放世界任务的组合角度理解这个问题。

我的第一句话总结：

> 机器人操作不是单个抓取动作，而是让机器人在不确定的真实世界里，通过感知、几何、规划、控制和接触交互，可靠改变物体状态。

## 章节结构

| 小节 | 我应该抓住的重点 |
| --- | --- |
| 1.1 Manipulation is more than pick-and-place | pick-and-place 是入口，不是边界；真实操作还包括整理、插入、滑动、折叠、推挤、利用接触等。 |
| 1.2 Open-world manipulation | 真实环境变化无限，泛化和鲁棒性比单次成功更重要。 |
| 1.3 Simulation | 仿真让学习和实验成本下降，但接触、传感器和现实迁移仍然是难点。 |
| 1.4 Interactive notes | 这门课的学习方式应该是“读概念 + 跑代码 + 提问题”。 |
| 1.5 Model-based design and analysis | 目标不是堆模块，而是系统化地组合 perception、planning、control。 |
| 1.6 Organization | 后续章节会按 manipulation building blocks 展开。 |
| 1.7 Exercises | 先熟悉 Drake 文档和交互式代码环境。 |

## 关键概念

- 操作任务的难点来自多模块耦合，而不是单个算法。
- 感知误差、接触不确定性和环境几何会同时影响动作结果。
- 模型驱动方法和数据驱动方法应该互补。
- “开放世界”意味着不能假设训练或测试覆盖了所有场景；系统需要在未见过的对象、厨房、光照、遮挡、摆放方式中保持合理行为。
- 仿真不是为了逃避真实世界，而是为了更快迭代算法、生成数据、排查系统问题，并在进入真机前降低风险。
- 模型化设计的价值在于：每个模块的输入输出、假设、失败模式都要清楚，否则 perception、planning、control 之间会互相甩锅。

## 本章给我的学习框架

我后续读每章时都用同一组问题检查：

1. 这一章解决 manipulation stack 的哪个部分？
2. 它假设哪些信息已经已知？
3. 它的输出会交给哪个下游模块？
4. 它最常见的失败模式是什么？
5. 有没有一个低依赖 Python 小案例能把核心概念跑通？

## 实现想法

- 建一个任务分类表：输入、目标、主要困难、可能算法。
- 后续每章都回到这个表，补充新的解决工具。
- Chapter 1 不需要代码案例；它负责定义为什么后续案例要覆盖 kinematics、pose estimation、grasp scoring、planning、control 和 RL。

## 未解决问题

- 哪些任务可以通过简单脚本完成，哪些任务必须引入规划或学习？
- 真实机器人中的接触误差通常如何被发现和纠正？
- 仿真中成功的 manipulation 策略，迁移到现实世界时最先坏在哪里？
- 如果系统失败了，如何判断问题来自感知、规划、控制、硬件还是任务定义？

## 本章学习结论

我不能把这门课学成“会调用几个库”。真正的目标是形成一个系统视角：看到一个操作任务时，能拆出对象状态、机器人状态、传感器输入、几何约束、运动规划、控制执行和失败恢复。这个仓库的案例集就是把这些模块逐步变成可运行的小实验。
